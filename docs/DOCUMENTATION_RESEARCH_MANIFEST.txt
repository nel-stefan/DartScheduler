API ENDPOINTS DISCOVERED: 27 Total

Players Management (6 endpoints):
  - POST   /api/import                       (bulk import from Excel)
  - GET    /api/players                      (list all)
  - PUT    /api/players/{id}                 (update)
  - DELETE /api/players/{id}                 (delete)
  - GET    /api/players/{id}/buddies         (get buddy preferences)
  - PUT    /api/players/{id}/buddies         (set buddy preferences)

Schedule Management (9 endpoints):
  - POST   /api/schedule/generate            (generate new schedule)
  - GET    /api/schedule                     (get latest/active)
  - GET    /api/schedules                    (list all schedules)
  - GET    /api/schedules/{id}               (get by ID with matches)
  - GET    /api/schedule/evening/{id}        (get single evening)
  - GET    /api/schedules/{id}/info          (schedule metadata)
  - PATCH  /api/schedules/{id}               (rename)
  - DELETE /api/schedules/{id}               (delete)
  - POST   /api/schedules/{id}/inhaal-avond  (add catch-up evening)
  - DELETE /api/schedules/{id}/evenings/{id} (delete evening)
  - POST   /api/schedules/import-season      (import historical data)

Scoring & Matches (2 endpoints):
  - PUT    /api/matches/{id}/score           (submit match result)
  - POST   /evenings/{id}/report-absent      (report player absent)

Statistics (2 endpoints):
  - GET    /api/stats                        (player standings)
  - GET    /api/stats/duties                 (duty assignments)

Evening Statistics (2 endpoints):
  - GET    /api/evenings/{id}/player-stats   (per-evening stats)
  - PUT    /api/evenings/{id}/player-stats/{playerId} (upsert)

Season Statistics (2 endpoints):
  - GET    /api/schedules/{id}/player-stats  (per-season stats)
  - PUT    /api/schedules/{id}/player-stats/{playerId} (upsert)

Export/Download (5 endpoints):
  - GET    /api/export/excel                 (full schedule as .xlsx)
  - GET    /api/export/pdf                   (full schedule as .pdf)
  - GET    /api/export/evening/{id}/excel    (evening form as .xlsx)
  - GET    /api/export/evening/{id}/pdf      (evening form as .pdf)
  - GET    /api/export/evening/{id}/print    (evening form as HTML)

System (1 endpoint):
  - GET    /api/system/logs                  (server logs)
  - GET    /health                           (health check)
  - GET    /api/config                       (app configuration)

DATA SCHEMAS IDENTIFIED: 12 Main Types

1. Player               - Member information
2. Schedule            - Competition schedule
3. Evening            - Single playing evening
4. Match              - Match between two players
5. PlayerStats        - Standings per player
6. DutyStats          - Secretary/counter assignments
7. EveningPlayerStat  - Per-evening statistics
8. SeasonPlayerStat   - Per-season statistics
9. ScheduleInfoResult - Schedule metadata for info page
10. SeasonSummary     - Lightweight schedule list item
11. GenerateScheduleRequest - Input for schedule generation
12. SubmitScoreRequest - Input for match result submission

================================================================================
KEY INSIGHTS
================================================================================

1. API is well-designed and RESTful
   - Follows HTTP method conventions
   - Proper status code usage
   - Clear resource hierarchy
   - Good error handling with sentinel errors

2. Data model is sophisticated
   - Supports complex match tracking (leg-by-leg)
   - Buddy preference management
   - Per-evening and per-season statistics
   - Catch-up evening mechanics

3. Documentation is basic but not bad
   - README covers main endpoints
   - Handler comments are minimal
   - No interactive features
   - No code examples beyond snippets
   - No error documentation

4. Architecture is clean
   - Clear separation of concerns
   - Well-organized packages
   - Good use of Go conventions
   - Repository pattern for data access

5. Security considerations
   - CORS is permissive (good for dev, needs tightening for prod)
   - No authentication required (good for closed LAN, needs API key for prod)
   - No rate limiting (needs for production)
